设计规范
=======

## 原则
- 保持简单 KISS
- 为分享最优质内容提供方便
- 注重信息长期价值
- 不断探索新方向 dare-to-dream-and-do
- user-friendly & eye-friendly
- 尽量nightly-build

## 团队
- 任何感兴趣的人都可参与策划&组织&创意&内容贡献&开发
- 所有文档和代码都托管于github，欢迎任何pull-request，如果需要可以在issue中申请直接提交权限


## 代码规范
- doc
- test
- code style

## 发布规范
- 测试和发布工作尽量做到自动化
- 发布之前添加tag，每周版本递增0.0.1
